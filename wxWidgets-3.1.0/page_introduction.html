什么是 wxWidgets ？

wxWidgets 是一个开源的C++框架，可以使用C++或者其他语言开发跨平台的、本地观感的 GUI 应用程序。

wxWidgets 最初是由 Julian Smart 在爱丁堡大学的人工智能应用研究所为内部使用而开发，并于1992年首次公布，1999年发布重大改进版本2。上一个大版本 3 是在 2013 年发布的。目前，wxWidgets由 Julian Smart, Vadim Zeitlin, Stefan Csomor, Robert Roebling, Vaclav Slavik 等许多人一起开发维护。

更多有关 wxWidgets的信息请移步 wxWidgets网站 https://www.wxwidgets.org 。

为什么选择 wxWidgets ？

相比于其他类似的库，wxWidgets的优点由：

  1. 唯一一个通过封装本地 GUI 部件而构建的框架库，所以在每一个平台上都有最好的用户体验。
  2. 仅仅使用标准 C++ 写成，不依赖任何的扩展或预处理。
  3. 开源、免费，无论是开源项目还是商业用途。

wxWidgets提供了简单、易学，却又非常丰富的 API。它同时又是成熟而稳定的，大约20年前使用 wxWidgets 2.0 的预发行版本写成的应用程序仍能够在今天的 3.0 版本下、几乎无需修改的构建。wxWidgets 拥有一个庞大的、活跃的、友好的社区，其中既有库的使用者，又有库的开发者。 它可以在很多语言中使用，包括 Python、Perl、Ruby、Lua、Haskell、D、Erlang、PHP，以用C++。

限于篇幅，在这里不能详尽介绍 wxWidgets，但是下面列出的是它的一些主要优点：

  • 在所有的主要桌面平台均可用。
  • 免费。
  • 开源，所以必要时很容易阅读和修改。
  • 超过100个示例程序。
  • 大量的文档（将近200000行）。
  • 直白的API。
  • 简单而强大的布局系统。
  • 可在运行时载入或在编译时嵌入的资源。
  • 灵活的事件系统。
  • 所有的常见和不常见的 GUI 控件。
  • 所有标准对话框。
  • 完全支持透明度、基于 2D 路径的绘图 API。
  • 内建支持许多文件格式（BMP、PNG、JPEG、GIF、XPM、PNM、PCX、TGA，……）。
  • 支持打印，帮助，剪贴板，拖放，文档/视图，……
  • 集成了平台原生的 HTML 渲染引擎。
  • 可停靠的窗口框架。
  • 类似处理器的 word 部件。
  • 带有语法高亮功能的、强大的文本编辑部件。
  • 还有很多，很多……

wxWidgets 的要求

wxWidgets 的第一级“端口”，即 wxWidgets API 的实现：

  • wxMSW：这是 Microsoft Windows 系统（从 XP 到 win10）的本地端口，包括 32 位和 64 位。主要支持的编译器是 Microsoft Visual C++ （支持版本从2003到2015，推荐至少2005版）以及 GNU 的 g++ （包括MinGW，TDM-GCC，MinGW-w64）。
  • wxGTK：wxGTK2 和 wxGTK3 分别是使用 GTK+ 库 2.x系列和 3.x 系列的端口。它们很类似，但是 wxGTK2 更成熟。两者都可以工作在几乎所有的 Unix 系统（Linux，FreeBSD，OpenBSD，NetBSD，Solaris，AIX，……）上，但是要求 GTK+ 2.6 以上或者 GTK+ 3.x。主要支持的编译器是 GNU g++。
  • wxOSX/Cocoa:：这是 Apple 电脑的本地端口。wxOSX/Cocoa 支持 32 位或 64 位的、运行 OS X 10.及以上版本的 英特尔 Mac 电脑。 端口可以使用 g++ 或 clang 构建。

其它平台 （比如iOS）、编译器（Windows上的 Borland C++ ，Unix 上的 Sun CC、 HP-UX aCC、 IBM xlC 或 SGI mipsPro）以及 端口 （wxQT，wxGTK1,，wxX11,，wxDFB……）也被支持，只是范围比较小。请查看平台详细说明页面以获取更多信息。

对 CPU 速度没有要求，但是你的 CPU(s) 速度越快，编译的就越快。要求要有足够的内存，尤其在 Windows 平台，要避免链接时用尽内存。依照以往的编译经验，你可能需要至少 4GB 的内存以应对链接。在所有的平台上都推荐有足够大的硬盘空间。确切的容量依赖于端口、编译器以及构建的配置，但是至少要 1GB ，可能需要更多。
